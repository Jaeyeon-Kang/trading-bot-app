내가(사용자) 지금까지 겪은 손실과 번아웃이 곧 너의 업무 범위니, 집중해라.

1. 프로젝트 목적  
   • 단타/스윙 트레이더용 “매수·매도 신호 알림 웹앱” MVP.  
   • 자동 주문은 아직 X.  
   • 종목 범위 = 주식·ETF(레버리지/인버스 포함)·지수·크립토까지 열어 둔다.  
   • 핵심 가치는 “지표-기반 숫자 + 간단한 GPT-뉴스 요약”만으로 **빠르고 싸게** 의사결정 돕기.

2. 필수 기능 (최소 동작 버전)  
   A. 데이터 수집  
      – 시세: Alpaca (지연) ⇒ 추후 Polygon 실시간(유료) 스위치 가능하게 인터페이스 분리.  
      – 뉴스·센티멘트: RSS / X 스크레이퍼.  
   B. 지표 계산  
      – Python + TA-Lib: EMA, RSI, 볼밴 등 기본 세트.  
      – 전략 JSON 스펙 예시  
        ```json  
        {  
          "symbol": "TSLA",  
          "rule": "ema_cross",  
          "params": { "fast": 12, "slow": 26 },  
          "stop": -5,  
          "take": 8  
        }  
        ```  
   C. 신호 발생 & 알림  
      – 조건 충족 시 Slack/Telegram Webhook.  
      – 포맷: `<종목> | <근거 지표> | SL/TP | 타임스탬프`.  
   D. GPT-4 모듈(옵션)  
      – 지난 24 h 뉴스 요약 + 긍/부정 점수만 반환.  
   E. 프런트  
      – Next.js + Tailwind, TradingView Widget 임베드.  
      – 화면 3개: 대시보드 / 전략 편집 / 로그.  
   F. 배포  
      – Docker Compose, Render 컨테이너(저가 티어) + Supabase (Postgres).  
      – .env 템플릿 포함.

3. 주의 사항  
   • 레버리지 ETF decay·괴리율 감시 로직 선택적 ON/OFF.  
   • 모든 시간 처리 UTC-5 (뉴욕 마감 16:00) 기준 고정.  
   • 백테스트 → 실시간 → 실거래 API (나중 단계) 순서로 단계적 확장 전제.  
   • 투자 판단 책임은 인간 사용자에게 귀속. 규제 회피용 면피 워딩은 최소화하되 넣어라.
